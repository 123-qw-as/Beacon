# 评价模型

## 适用场景

对多个方案/对象按多指标综合打分排序。典型竞赛题：选最优方案（供应商/选址/算法）、
风险等级评定、综合实力排名。判别特征：有"评价""排序""综合""优劣""权重"等关键词，
且指标多于 3 个、方案多于 2 个。常用方法：AHP（层次分析法）、TOPSIS、熵权法、模糊综合评价。

## 核心公式

### AHP（层次分析法）
1. 构造判断矩阵 A = (a_ij)，a_ij = 指标 i 相对指标 j 的重要性（1-9 标度）。
2. 求权重 w：A w = λ_max w，w 归一化（最大特征值对应特征向量）。
3. 一致性检验：CI = (λ_max - n)/(n-1)，CR = CI/RI。CR < 0.1 通过，否则调整判断矩阵。
   RI 查表：n=3→0.58, n=4→0.90, n=5→1.12, n=6→1.24, n=7→1.32, n=8→1.41。

### TOPSIS（逼近理想解排序）
1. 标准化决策矩阵 X → Z（向量归一化或极差归一化）。
2. 加权：V = Z · diag(w)（w 来自 AHP 或熵权法）。
3. 正理想解 v+ = max(v_ij)，负理想解 v- = min(v_ij)（成本型指标取反）。
4. 各方案到 v+/v- 的距离 D+/D-，贴近度 C_i = D- / (D+ + D-)。C_i 越大越优。

### 熵权法（客观赋权）
1. 标准化后计算比重 p_ij = x_ij / Σ_i x_ij。
2. 熵值 e_j = -k Σ_i p_ij ln(p_ij)，k = 1/ln(n)。
3. 权重 w_j = (1 - e_j) / Σ_j (1 - e_j)。熵越小（差异越大）权重越高。

## 建模步骤

1. 明确指标体系：分效益型（越大越好）与成本型（越小越好），标注方向。
2. 赋权：主观（AHP，依赖专家判断）+ 客观（熵权，依赖数据离散度）可组合用。
3. 评分：TOPSIS 或加权求和；多方法交叉验证排序稳定性。
4. 敏感性：扰动权重 ±20%，看排名是否翻转；翻转说明结论对权重敏感，需说明。

## 常见陷阱

- AHP 不做一致性检验：CR > 0.1 的判断矩阵无效，评委必查。这是 AHP 的命门。
- 成本型指标没取反：TOPSIS 正理想解对成本型应取 min，直接取 max 会排序颠倒。
- 熵权法遇零值：p_ij=0 时 ln(0) 无定义，需加小常数 ε 平滑（p_ij + ε）。
- 主客观权重直接平均：AHP 权重和熵权权重直接算术平均无理论依据，
  应用乘法合成 w_j = (w_subj · w_obj) / Σ 或最小信息熵组合。
- 指标共线性：高度相关的指标重复计权，应先做相关分析或主成分降维。
