# 排队论模型

## 适用场景

资源有限的服务系统：顾客到达需排队等待服务。典型竞赛题：交通调度（信号灯/收费站）、
内涝排水容量评估（排水口服务降雨）、共享单车调度（站点容量约束）、银行/医院窗口配置。
判别特征：有"到达率""服务率""系统容量""等待时间"等关键词，且资源是离散可数的服务台。

## 核心公式

M/M/c 模型：到达服从 Poisson(λ)，服务服从指数(μ)，c 个服务台。

- 系统负荷强度：ρ = λ / (c μ)，稳态要求 ρ < 1（否则队列无限增长）
- 系统空闲概率：P0 = [Σ_{k=0}^{c-1} (λ/μ)^k / k! + (λ/μ)^c / (c! (1-ρ))]^{-1}
- 平均队列长度：Lq = P0 · (λ/μ)^c · ρ / (c! · (1-ρ)^2)
- 平均等待时间：Wq = Lq / λ（Little 定律）
- 系统内平均人数：Ls = Lq + λ/μ

M/M/1 是 c=1 特例：Ls = ρ/(1-ρ)，Ws = 1/(μ-λ)。

## 建模步骤

1. 确认到达过程是否近似 Poisson：画到达间隔的指数 Q-Q 图，或 χ² 检验。
2. 确认服务时间是否近似指数：若服务时间方差远小于均值（确定性服务），改用 M/D/c。
3. 计算 ρ，验证 ρ < 1；若 ρ ≥ 1 说明系统不稳定，需增加服务台或限流。
4. 代入稳态公式求 Lq、Wq；若题问"顾客等待超 T 分钟的概率"，用 Erlang C 公式。
5. 多服务台异构（服务率不同）时，解析解复杂，转仿真（离散事件模拟）。

## 常见陷阱

- 忘记验证 ρ < 1：ρ ≥ 1 时稳态公式无意义，Lq 算出负数或发散。
- 混淆 Lq（队列长，不含服务中）与 Ls（系统长，含服务中）：题目问"排队人数"是 Lq。
- 服务台数 c 选错：题目给"3 个窗口"但其中 1 个是 VIP 专用，则对普通顾客 c=2。
- 到达率单位不一致：λ 用"人/分钟"但 μ 用"人/小时"，需统一。
- 评委常查一致性：若声称 M/M/c，须给出到达/服务分布的检验，不能只口头假设。
