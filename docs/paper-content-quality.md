# 竞赛论文内容深度与篇幅门禁

本文说明 Beacon 如何避免“PDF 编译成功但正文只有 8--10 页”，以及如何复现当前绿色物流参考稿。
它描述现行代码契约；历史方案仍保存在 `docs/superpowers/`，不得用来判断当前行为。

## 1. 验收目标

论文必须同时满足内容证据和版式篇幅要求：

- 主方案真实读取声明的附件，并通过 RESULT 数值合理性、有限车队和服务覆盖检查；
- 至少存在两个独立执行且口径一致的基线；
- 敏感性中心点复现正式主方案，不混入旧 attempt；
- 城市绿色物流主方案必须输出算法搜索、随机稳健性、服务诊断和五类动态事件证据；
- PDF 在真正的附录首页前至少有 20 个非空正文页、15000 个非空白字符；
- LaTeX 两遍编译成功，且逐页渲染后无乱码、裁切、重叠或严重溢出。

默认篇幅阈值由以下环境变量控制：

```dotenv
MATH_AGENT_MIN_PAPER_BODY_PAGES=20
MATH_AGENT_MIN_PAPER_BODY_CHARS=15000
```

目录中的“附录”条目不算附录首页。正文起点和附录起点都按页面首个非空标题识别，避免目录误判。

## 2. 证据生成链

绿色物流安全求解器在一次正式主方案执行中完成以下工作：

1. 从订单、距离矩阵、时间窗和客户坐标四个附件构造有限异构车队方案；
2. 对构造解执行路线内 2-opt，并记录初末目标、改善量、移动次数和运行时间；
3. 固定正式路线，用题面速度分布生成 200 个可复现交通情景；
4. 在每个情景中重算等待、晚到、燃油或电耗、返程能耗和碳成本；
5. 统计晚到任务、尾部晚到、载重/容积利用率和空载返程比例；
6. 独立构造订单取消、新增订单、地址变更、时间窗收紧和车辆故障事件；
7. 输出结构化证据行，供 writer、evaluation 和 finalizer 共用。

四类深度证据标签为：

```text
ALGORITHM_SEARCH
ROBUSTNESS
SERVICE_DIAGNOSTICS
DYNAMIC_EVENTS
```

门禁不仅检查标签存在，还检查必需字段、概率范围、非负性和交叉一致性。例如算法改善量必须等于
初末目标之差，五类事件成功率必须与总体回退率一致。缺字段、非法概率或自相矛盾的输出不能进入论文。

## 3. 动态事件口径

- 订单取消：从原路线移除任务后复算可行性；
- 新增订单：复制一个真实附件任务作为压力样本，枚举全部路线，不虚构需求属性；
- 地址变更：保留原时间窗，只替换为附件中的另一个真实客户节点；
- 时间窗收紧：先移除旧任务，再用收紧后的时间窗唯一回插任一路线；
- 车辆故障：释放整条故障路线任务，并在其余车辆中累计迁移。

地址和时间窗变更不能保留旧任务再插入一个副本。新增订单也没有“来源路线”，不得无理由排除任一
候选路线。局部恢复失败的事件计入回退率，不能从统计中删除。

## 4. 写作与 PDF 门禁

writer 只消费通过验证的结构化证据，并在摘要、模型求解、敏感性、模型评价和结论中分别解释：

- 2-opt 带来的构造解改善；
- 固定路线蒙特卡洛与单因素重新决策的区别；
- 服务违约尾部和车辆资源利用情况；
- 五类动态事件的适用边界与升级策略。

表格转换使用无段首缩进的 `tabularx`，长敏感性数组独立成段，避免把内容挤出页边界。finalizer 同时
检查正文页数、非空页和字符量；放大字号、增加空白页或延长代码附录都不能通过门禁。

## 5. 参考重建

从一个已经包含题面、附件摘要、模型和评审状态的验证运行重建内容增强参考稿：

```powershell
uv run python scripts/render_content_optimized_reference.py `
  --source-state runs/verified-run/final_state.json `
  --data-dir runs/verified-run/data `
  --out runs/content-optimized-reference
```

输出目录必须为空。脚本会重新执行主方案、三个基线和敏感性扫描，复用正式数值、数据血缘和深度证据
门禁，然后生成 Markdown、TeX、PDF 与 `final_state.json`。它不会启动 supervisor 或生成新的
`completion.json`，因此是可复现参考验证，不是新的完整 CLI 闭环。

2026-07-17 的代表性结果位于 `runs/wodecesium-content-optimized-final-v2-20260717`：PDF 共 31 页，
正文 25 页、25057 个非空白字符；Python 全量回归为 586 passed / 4 skipped。该运行的主方案成本为
140450.14 元，2-opt 目标改善率为 13.5361%，并执行了 200 个随机交通情景和 50 个动态事件情景。

## 6. 解释边界

当前结果仍有以下边界，论文必须如实陈述：

- 构造加 2-opt 是启发式可行上界，没有分支定界或最优性间隙；
- 蒙特卡洛固定正式路线评估交通扰动，不等于随机规划或机会约束优化；
- 动态实验是独立单事件压力测试，不代表连续多事件滚动系统的长期最优性；
- 客户级诊断尚未按地理分区或服务等级聚类；
- 参考重建不能替代 paper critic、evaluation、human review 和最终 completion 校验。
